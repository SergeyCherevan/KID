# Выполнение плана новой функции

## Контекст
Выполняет план реализации новой функции, созданный командой /generate-plan-for-new-feature. Выполняет задачи последовательно, проверяя результат на каждом этапе.

## Задача
Выполни план реализации функции, следуя предоставленному плану.

### Процесс выполнения

1. **Анализ плана**
   - Прочитай предоставленный план
   - Убедись, что понимаешь все задачи
   - Проверь зависимости между задачами

2. **Последовательное выполнение задач**
   Для каждой задачи из плана:
   - Выполни задачу согласно описанию
   - Создай или измени необходимые файлы
   - Следуй существующему стилю кода в проекте
   - Добавь необходимые комментарии и XML-документацию

3. **Создание новых компонентов**
   При создании новых компонентов:
   - Создай интерфейс в соответствующей папке Interfaces/
   - Создай реализацию, следуя паттернам проекта
   - Зарегистрируй в DI контейнере (ServiceCollectionExtensions)
   - Добавь XML-комментарии ко всем публичным членам

4. **Изменение существующих компонентов**
   При изменении существующих компонентов:
   - Сохрани обратную совместимость где возможно
   - Следуй принципам SOLID
   - Не нарушай существующую функциональность
   - Обнови документацию если необходимо

5. **Локализация**
   Если добавлены новые строки интерфейса:
   - Добавь ключи в Strings.ru-RU.resx
   - Добавь ключи в Strings.en-US.resx
   - Добавь ключи в Strings.uk-UA.resx
   - Используй формат Category_Name для ключей

6. **Документация**
   Если функция требует обновления документации:
   - Обнови соответствующие .md файлы в docs/
   - Добавь примеры использования
   - Обнови описание архитектуры если изменилась структура

7. **Проверка результата**
   После выполнения всех задач:
   - Проверь, что код компилируется без ошибок
   - Проверь, что нет ошибок линтера
   - Убедись, что все зависимости правильно зарегистрированы
   - Проверь, что локализация добавлена для всех языков

## Требования
- Выполняй задачи последовательно, не пропуская шаги
- После каждой задачи проверяй результат
- Следуй существующему стилю кода проекта
- Используй DI для всех зависимостей
- Все строки интерфейса должны быть локализованы
- Все публичные методы должны иметь XML-комментарии
- Код должен следовать принципам SOLID
- Обрабатывай ошибки с try-catch где необходимо
- Используй async/await для асинхронных операций

## Отчёт о выполнении
После выполнения всех задач предоставь отчёт:
- Список созданных файлов
- Список изменённых файлов
- Список добавленных ключей локализации
- Обновлённые файлы документации
- Замечания и рекомендации
