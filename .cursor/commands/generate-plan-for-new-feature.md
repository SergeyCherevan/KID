# Генерация плана для новой функции

## Контекст
Создаёт детальный план реализации новой функции в проекте .KID. План включает анализ требований, архитектурные решения, список задач и оценку сложности.

## Задача
Создай детальный план реализации функции: {{featureName}}

План должен включать:

### 1. Анализ требований
- Описание функции и её цели
- Целевая аудитория и сценарии использования
- Входные и выходные данные
- Ограничения и требования

### 2. Архитектурный анализ
- Какие подсистемы затронуты
- Какие новые компоненты нужно создать
- Какие существующие компоненты нужно изменить
- Зависимости между компонентами

### 3. Список задач
Разбей реализацию на конкретные задачи:
- Создание новых файлов (с указанием путей)
- Изменение существующих файлов
- Добавление локализации
- Обновление документации
- Тестирование

### 4. Порядок выполнения
Упорядочь задачи в логическом порядке выполнения

### 5. Оценка сложности
Для каждой задачи укажи:
- Сложность (низкая/средняя/высокая)
- Примерное время выполнения
- Потенциальные риски

## Требования
- План должен быть конкретным и выполнимым
- Учитывай существующую архитектуру проекта (MVVM, DI, локализация)
- Учитывай, что проект использует WPF, .NET 8.0, C#
- План должен следовать принципам SOLID
- Учитывай необходимость локализации всех строк интерфейса
- Учитывай необходимость обновления документации

## Формат вывода
Выведи план в структурированном виде с нумерацией и подзадачами, чтобы его можно было использовать для команды /execute-plan-for-new-feature
