# План: генерация `docs/ProjectTemplates.md`

## 1. Анализ требований

1.1. **Что делаем**

- Сформировать файл документации: `docs/ProjectTemplates.md`.
- Описать **все** шаблоны/скрипты из `[KID.WPF.IDE/ProjectTemplates](KID.WPF.IDE/ProjectTemplates)`.

1.2. **Цель**

- Дать преподавателю/ученику и разработчику IDE единый справочник по шаблонам: **что делает каждый скрипт**, чему учит, какие API использует, какие клавиши/ввод задействованы, какая структура и какие алгоритмы внутри.

1.3. **Целевая аудитория и сценарии**

- Ученики: выбор подходящего примера для темы (циклы, ввод, графика, звук, события/опрос состояния мыши/клавиатуры).
- Преподаватели: быстро подобрать демонстрацию и объяснить алгоритм.
- Разработчики .KID: понимать назначение/покрытие примеров и поддерживать их согласованность между локалями.

1.4. **Входные данные**

- Файлы шаблонов `.cs` из:
  - `[KID.WPF.IDE/ProjectTemplates/ru-RU](KID.WPF.IDE/ProjectTemplates/ru-RU)`
  - `[KID.WPF.IDE/ProjectTemplates/en-US](KID.WPF.IDE/ProjectTemplates/en-US)`
  - `[KID.WPF.IDE/ProjectTemplates/uk-UA](KID.WPF.IDE/ProjectTemplates/uk-UA)`
- Механика загрузки шаблона в IDE:
  - `[KID.WPF.IDE/Services/Initialize/WindowConfigurationService.cs](KID.WPF.IDE/Services/Initialize/WindowConfigurationService.cs)`
  - `[KID.WPF.IDE/Services/Initialize/WindowConfigurationData.cs](KID.WPF.IDE/Services/Initialize/WindowConfigurationData.cs)`
  - `[KID.WPF.IDE/DefaultWindowConfiguration.json](KID.WPF.IDE/DefaultWindowConfiguration.json)`
  - (как используется при «Новый файл») `[KID.WPF.IDE/ViewModels/MenuViewModel.cs](KID.WPF.IDE/ViewModels/MenuViewModel.cs)`

1.5. **Выходные данные**

- Новый документ `docs/ProjectTemplates.md`.
- (Опционально, но рекомендуется) добавить ссылку на него в `[docs/README.md](docs/README.md)`.

1.6. **Ограничения/требования**

- Документ пишем **на русском**.
- Описание должно быть «полным»: **поведение**, **структуры/переменные**, **методы/функции**, **алгоритмы**, **управление (клавиши/мышь/ввод)**, **вывод (консоль/графика/музыка)**.
- Учитывать, что в шаблонах есть 3 локали; нужно явно описать, что отличается (обычно строки/комментарии), а что совпадает (логика/алгоритм).

2.1. **Какие подсистемы затронуты**

- Только документация (`docs/`).

2.2. **Новые компоненты**

## 3. Список задач

3.1. **Инвентаризация шаблонов** (все 39 файлов)

- Сгруппировать «семейства» шаблонов по относительному пути без локали (пример: `Gold/BouncingBall.cs`) и для каждого семейства зафиксировать:
  - пути для `ru-RU/en-US/uk-UA`
  - совпадает ли логика (обычно да) и чем отличаются тексты.

3.2. **Определить единый формат карточки шаблона**
Для каждого шаблона в `ProjectTemplates.md` завести секцию со стабильными полями:

- Назначение (1–2 абзаца)
- Чему учит (список)
- Поведение по шагам (как выполняется программа)
- Ввод и управление (клавиши/мышь/Console.ReadLine)
- Вывод (консоль/графика/музыка)
- Ключевые структуры данных/переменные (что за что отвечает)
- Ключевые алгоритмы (например: цикл, расчёт `dt`, отскоки, проверка попадания, стёрка по дистанции до сегмента)
- Используемые API KID (ссылки на docs)
- Локали и различия

3.3. **Сгенерировать каркас `docs/ProjectTemplates.md**`

- Создать оглавление и разделы по категориям:
  - Базовые: `HelloWorld.cs`, `Demo1.cs`, `Demo1.FullStructure.cs`
  - `CheatSheets/`: `GameLoop.Template.cs`, `SafeInput.Template.cs`
  - `Gold/`: `BouncingBall.cs`, `CatchTheCircle.cs`, `GuessNumber.Console.cs`, `KeyboardPiano.cs`, `MousePainter.cs`
  - `NextLevel/`: `Generative.SpiralOrFractal.cs`, `Particles.Firework.cs`, `Scenes.StateMachine.cs`
- Вставить под каждую карточку минимальные факты (пути файлов, типы ввода/вывода, извлечённые «первые комментарии» если есть).

3.4. **Написать «полные» описания для каждого шаблона**

- Пройти по шаблонам (рекомендуемо брать за основу `ru-RU` версию) и дописать:
  - подробный разбор алгоритмов
  - объяснение ключевых блоков кода
  - список возможных упражнений/вариантов модификации (коротко, но по делу)

3.5. **Интеграция в документацию** (опционально)

- Обновить `[docs/README.md](docs/README.md)`: добавить ссылку на `ProjectTemplates.md` в список документации.
